# 系统总体架构

## 1. 架构目标
构建一个可扩展、可审计、可集成的企业级签证业务平台。

## 2. 推荐架构
现阶段采用模块化单体，后续按业务负载和组织边界逐步拆分。

## 3. 核心模块
- 身份与权限
- 租户与组织
- 客户与申请人档案
- 案件管理
- 文档接收与存储
- 工作流引擎
- 规则引擎
- 审核与审批
- 通知中心
- 计费与订阅
- 审计与可观测性

## 4. 技术建议
- 前端：Next.js + TypeScript
- 后端：NestJS 或 Fastify
- 数据库：PostgreSQL
- 缓存与队列：Redis
- 文件存储：S3 兼容对象存储
- 监控：日志、指标、链路追踪三件套

## 5. 关键原则
- 所有状态流转可审计
- 所有外部调用可重试
- 所有关键操作有权限边界
- 所有敏感数据默认高保护
